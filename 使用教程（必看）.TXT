实训报告生成器使用教程（必看）

一、这个工具是干什么的

本工具用于根据课堂 TXT 笔记自动生成实训报告 Word 文档。

它支持：
1. 在网页里拖入多个 TXT 文件。
2. 自动按文件名中的“月-日”合并同一天内容。
3. 自动生成课程目标、课程内容、课程内容详情、课程代码及执行过程。
4. 基于实训报告 Word 模板生成 .docx 文件。
5. 自动生成 PyCharm 运行结果风格图片，并放到每个任务代码下方。
6. 默认对目标、详情、任务说明和示例代码做差异化处理，尽量降低多篇报告雷同。
7. 可选使用 DeepSeek 等 OpenAI 兼容 API 进行 AI 增强生成。


二、下载项目

方式一：直接下载 ZIP

1. 打开项目地址：
   https://github.com/Sky-Hub-aurora/Sky

2. 点击绿色的 Code 按钮。

3. 点击 Download ZIP。

4. 解压到本地文件夹。


方式二：使用 git clone

在命令行运行：

git clone https://github.com/Sky-Hub-aurora/Sky.git
cd Sky


三、安装运行环境

1. 先安装 Python。

建议安装 Python 3.10 或以上版本。

下载地址：
https://www.python.org/downloads/

安装 Python 时建议勾选：
Add python.exe to PATH

2. 安装依赖。

在项目文件夹里打开命令行，运行：

pip install -r requirements.txt

如果 pip 命令不能用，可以尝试：

python -m pip install -r requirements.txt


四、放置 Word 模板

必须把实训报告模板 Word 文档放到项目文件夹里。

推荐命名为：

实训报告（过程）模板.docx

如果模板文件名不同，也可以在网页里的“模板路径”输入框中填写完整路径。

注意：
1. 模板是生成报告的基底。
2. 工具默认保留模板第一页封面。
3. 后面的课程报告正文由工具自动生成。


五、启动工具

Windows 用户推荐直接双击：

启动实训报告网页.bat

这个 bat 会先自动安装 requirements.txt 里的依赖，再启动网页服务。

如果双击没有反应，可以在项目目录打开命令行，运行：

python 实训报告生成器网页.py

启动成功后，浏览器访问：

http://127.0.0.1:8765/


六、TXT 文件命名规则

TXT 文件名建议使用：

月-日-次.txt

例如：

7-4-01.txt
7-4-02.txt
7-5-01.txt

工具会按照“月-日”自动合并同一天内容。

例如：

7-4-01.txt
7-4-02.txt

这两个会合并成第一天。

7-5-01.txt

会作为第二天。

也就是说，前两个数字一样，就认为是同一天。


七、网页怎么使用

1. 打开网页：

http://127.0.0.1:8765/

2. 把 TXT 文件拖到左侧“TXT 输入窗口”。

3. 填写输出 Word 文件名。

推荐格式：

班级-组号-学号-姓名.docx

4. 可根据需要填写：

班级
组别
学号
姓名
学院
专业班级

5. 如果需要填写模板第一页封面，勾选“填写封面”。

6. 如果希望每天之间分页，勾选“每天分页”。

7. 点击“生成 Word”。

8. 生成完成后，在右侧“产出窗口”点击“下载 Word”。

说明：
1. 代码下方的运行结果图会自动生成，不需要你再手动截图。
2. 如果你自己准备了真实运行截图，命令行版本仍然支持通过 --screenshots 指定截图文件夹。
3. 自动运行结果图会尽量执行代码并捕获输出；遇到复杂或不安全代码时，会根据代码内容推断一份合理运行结果。


八、AI 增强怎么用

AI 增强不是必须的。

不勾选 AI 增强时，工具会使用本地规则生成报告，不需要联网。

如果要使用 DeepSeek API：

1. 勾选“AI 增强”。

2. 点击“填入 DeepSeek 预设”。

会自动填入：

AI API 地址：https://api.deepseek.com
模型名：deepseek-v4-flash
温度：0.2

3. 在 API Key 输入框中填写你的 DeepSeek API Key。

4. 点击“生成 Word”。

注意：
1. API Key 需要你自己到 DeepSeek 控制台获取。
2. API Key 不会保存到项目文件中，只在本次生成时发送给本机服务使用。
3. 如果 API 额度不足、Key 错误、网络不通，AI 增强会失败。
4. AI 增强失败时，可以关闭 AI 增强，继续使用本地规则生成。


九、常见问题

问题 1：双击启动文件没反应怎么办？

答：
在项目文件夹空白处打开命令行，运行：

python 实训报告生成器网页.py

看命令行里有没有报错。


问题 2：提示没有 python 命令怎么办？

答：
说明 Python 没安装好，或者安装时没有加入 PATH。
重新安装 Python，并勾选 Add python.exe to PATH。


问题 3：提示缺少 python-docx 或 Pillow 怎么办？

答：
在项目文件夹运行：

pip install -r requirements.txt


问题 3-1：提示 No module named 'cgi' 怎么办？

答：
这是旧版本脚本在 Python 3.13 上可能出现的问题，因为 Python 3.13 删除了 cgi 模块。
请重新从 GitHub 下载最新版本，或者执行 git pull 更新项目。
最新版已经不依赖 cgi，不需要安装 legacy-cgi。


问题 4：生成失败，提示找不到模板怎么办？

答：
把 Word 模板放到项目文件夹，并命名为：

实训报告（过程）模板.docx

或者在网页中的“模板路径”填写模板完整路径。


问题 5：TXT 没有按天合并怎么办？

答：
检查文件名是否符合类似格式：

7-4-01.txt
7-4-02.txt
7-5-01.txt

同一天的文件，前两个数字要一致，例如 7-4。


问题 6：DeepSeek API 用不了怎么办？

答：
优先检查：

1. 是否勾选了 AI 增强。
2. 是否点击了“填入 DeepSeek 预设”。
3. API 地址是否为：https://api.deepseek.com
4. 模型名是否为：deepseek-v4-flash
5. API Key 是否正确。
6. DeepSeek 账户是否还有额度。
7. 如果出现 WinError 10013，通常是 Windows 防火墙、杀毒软件、代理/VPN 或当前运行环境禁止 Python 访问外网。请关闭当前服务窗口后，双击“启动实训报告网页.bat”重新启动；如果仍失败，请在防火墙中允许 python.exe 联网，并检查代理/VPN 设置。

如果仍然不能用，可以先关闭 AI 增强，用本地规则生成。

实训报告生成器使用教程（必看）

一、这个工具是干什么的

本工具用于根据课堂 TXT 笔记自动生成实训报告 Word 文档。

它支持：
1. 在网页里拖入多个 TXT 文件。
2. 自动按文件名中的“月-日”合并同一天内容。
3. 自动生成课程目标、课程内容、课程内容详情、课程代码及执行过程。
4. 基于实训报告 Word 模板生成 .docx 文件。
5. 如果 TXT 中没有明确写出任务或练习，第四小专题不会自动编造代码，只会写提示语。
6. 自动生成 PyCharm 运行结果风格图片，并放到每个任务代码下方。
7. 生成 Word 的同时生成 xlsx 对照表，用于核对 TXT 原文和报告内容。
8. 默认对目标、知识点名称、内容详情、任务说明和示例代码做深度差异化处理，尽量降低多篇报告雷同。
9. 可选使用 DeepSeek 等 OpenAI 兼容 API 进行 AI 增强生成，API Key 可以直接在网页隐藏输入框里填写，并自动保存到 .env。


二、下载项目

方式一：直接下载 ZIP

1. 打开项目地址：
   https://github.com/Sky-Hub-aurora/Sky

2. 点击绿色的 Code 按钮。

3. 点击 Download ZIP。

4. 解压到本地文件夹。


方式二：使用 git clone

在命令行运行：

git clone https://github.com/Sky-Hub-aurora/Sky.git
cd Sky


三、安装运行环境

1. 先安装 Python。

建议安装 Python 3.10 或以上版本。

下载地址：
https://www.python.org/downloads/

安装 Python 时建议勾选：
Add python.exe to PATH

2. 安装依赖。

在项目文件夹里打开命令行，运行：

pip install -r requirements.txt

如果 pip 命令不能用，可以尝试：

python -m pip install -r requirements.txt


四、放置 Word 模板

必须把实训报告模板 Word 文档放到项目文件夹里。

推荐命名为：

实训报告（过程）模板.docx

如果模板文件名不同，也可以在网页里的“模板路径”输入框中填写完整路径。

注意：
1. 模板是生成报告的基底。
2. 工具默认保留模板第一页封面。
3. 后面的课程报告正文由工具自动生成。


五、启动工具

Windows 用户推荐直接双击：

启动实训报告网页.bat

这个 bat 会先自动安装 requirements.txt 里的依赖，再启动网页服务。

如果双击没有反应，可以在项目目录打开命令行，运行：

python 实训报告生成器网页.py

启动成功后，浏览器访问：

http://127.0.0.1:8765/


六、TXT 文件命名规则

TXT 文件名建议使用：

月-日-次.txt

例如：

7-4-01.txt
7-4-02.txt
7-5-01.txt

工具会按照“月-日”自动合并同一天内容。

例如：

7-4-01.txt
7-4-02.txt

这两个会合并成第一天。

7-5-01.txt

会作为第二天。

也就是说，前两个数字一样，就认为是同一天。


七、网页怎么使用

1. 打开网页：

http://127.0.0.1:8765/

2. 把 TXT 文件拖到左侧“TXT 输入窗口”。

3. 填写输出 Word 文件名。

推荐格式：

班级-组号-学号-姓名.docx

4. 可根据需要填写：

班级
组别
学号
姓名
学院
专业班级

5. 如果需要填写模板第一页封面，勾选“填写封面”。

6. 如果希望每天之间分页，勾选“每天分页”。

7. 点击“生成 Word”。

8. 生成完成后，在右侧“产出窗口”点击“下载 Word”和“下载对照表”。

说明：
1. 只有 TXT 中明确出现“练习”或“任务”时，才会生成代码和运行结果图。
2. 如果当天没有任务或练习，第四小专题会写：“这次没有任务和练习，可能老师上课布置的作业，回忆回忆，自己添加哦！”
3. 代码下方的运行结果图会自动生成，不需要你再手动截图。
4. 如果你自己准备了真实运行截图，命令行版本仍然支持通过 --screenshots 指定截图文件夹。
5. 自动运行结果图会尽量执行代码并捕获输出；遇到复杂或不安全代码时，会根据代码内容推断一份合理运行结果。
6. xlsx 对照表命名为“对照表”，分为“项目、原txt文档、生成的word文档”三列；没有任务时，“任务与练习”和“代码”项目对应内容为空。


八、AI 增强怎么用

AI 增强不是必须的。

不勾选 AI 增强时，工具会使用本地多样化规则生成报告，不需要联网。

AI 接口使用 OpenAI 兼容的 /chat/completions 格式。
除了 DeepSeek，也可以尝试 Kimi、通义千问、智谱、OpenRouter、硅基流动等兼容接口。
现在 API Key 可以直接在网页里填写，输入框会隐藏显示；网页会先做格式检查，再自动写入项目根目录的 .env。下次启动时会自动读取保存配置。
如果某些接口不支持 response_format，程序会自动换普通 JSON 提示重试。

最方便的配置方法：

1. 勾选“AI 增强”。

2. 在“通道预设”里选择 DeepSeek、OpenRouter、Kimi、通义千问、智谱或硅基流动。

3. 点击“套用接口预设”，网页会自动填写 API 地址、模型名和温度。

4. 把你的 API Key 粘贴到“API Key（隐藏输入）”框里，输入内容会像密码一样隐藏，且会自动保存到 .env。

5. 也可以点击“保存 AI 配置”手动保存。

6. 如果出现 WinError 10013，先点击“检测代理”，再点击“检测全部通道”，确认每个 API 地址是否能通过系统代理或 Clash/V2Ray 访问。

7. 点击“测试 AI 连接”，成功后再生成 Word。

8. 如果下次不想让别人继续用你的 Key，点击“清除 Key”，它会同时清空网页输入和 .env 中的 AI_API_KEY。

可选备用配置：

如果你不想每次都在网页里填写，也可以复制 .env.example 文件，把复制出来的文件改名为：

.env

然后在 .env 中填写：

AI_API_KEY=你的API密钥
AI_PROVIDER=deepseek
AI_BASE_URL=接口地址
AI_MODEL=模型名称
AI_TEMPERATURE=0.6
AI_TIMEOUT=60
AI_PROXY_URL=

填完 .env 后，关闭当前网页服务窗口，重新启动“启动实训报告网页.bat”。

如果你已经在 Windows 防火墙里允许 python.exe 联网，但仍然出现 WinError 10013：

1. 看网页日志里显示的“当前 Python”路径，确认防火墙放行的是同一个 python.exe。
2. 点击“检测代理”，工具会自动寻找系统代理和常见本地代理端口。
3. 点击“检测全部通道”，如果某个通道返回 HTTP 401/403/405，也说明网络已经到达接口，后续主要检查 API Key、模型名和额度。
4. 如果你使用 Clash、V2Ray、代理软件或校园网代理，也可以在网页“代理地址（可选）”里填写代理地址；如果使用 .env，也可以写到 AI_PROXY_URL。
5. 常见代理地址示例：

http://127.0.0.1:7890
http://127.0.0.1:10809

6. 填好网页配置或 .env 配置后，点击“测试 AI 连接”。
7. 如果测试成功，再点击“生成 Word”。

如果要使用 DeepSeek API：

1. 勾选“AI 增强”。

2. 在通道预设里选择“DeepSeek”，再点击“套用接口预设”，网页会自动填成类似：

AI API 地址：https://api.deepseek.com
模型名：deepseek-chat
温度：0.6

3. 在“API Key（隐藏输入）”里填写你的 DeepSeek API Key。

4. 点击“生成 Word”。

注意：
1. API Key 需要你自己到 DeepSeek 控制台获取。
2. 如果你使用 .env，.env 已加入 .gitignore，不要把 .env 上传到 GitHub。
3. 如果 API 额度不足、Key 错误、网络不通，AI 增强会失败。
4. AI 增强失败时，可以关闭 AI 增强，继续使用本地规则生成。
5. 网页新增“AI 输入框”和“发送到 AI”，它会请求后端 /api/chat，再由后端携带网页保存到 .env 的 Key 调用真实模型。


九、常见问题

问题 1：双击启动文件没反应怎么办？

答：
在项目文件夹空白处打开命令行，运行：

python 实训报告生成器网页.py

看命令行里有没有报错。


问题 2：提示没有 python 命令怎么办？

答：
说明 Python 没安装好，或者安装时没有加入 PATH。
重新安装 Python，并勾选 Add python.exe to PATH。


问题 3：提示缺少 python-docx 或 Pillow 怎么办？

答：
在项目文件夹运行：

pip install -r requirements.txt


问题 3-1：提示 No module named 'cgi' 怎么办？

答：
这是旧版本脚本在 Python 3.13 上可能出现的问题，因为 Python 3.13 删除了 cgi 模块。
请重新从 GitHub 下载最新版本，或者执行 git pull 更新项目。
最新版已经不依赖 cgi，不需要安装 legacy-cgi。


问题 4：生成失败，提示找不到模板怎么办？

答：
把 Word 模板放到项目文件夹，并命名为：

实训报告（过程）模板.docx

或者在网页中的“模板路径”填写模板完整路径。


问题 5：TXT 没有按天合并怎么办？

答：
检查文件名是否符合类似格式：

7-4-01.txt
7-4-02.txt
7-5-01.txt

同一天的文件，前两个数字要一致，例如 7-4。


问题 6：DeepSeek API 用不了怎么办？

答：
优先检查：

1. 是否勾选了 AI 增强。
2. 是否选择了对应通道，并点击了“套用接口预设”。
3. API 地址是否为：https://api.deepseek.com
4. 模型名是否为：deepseek-chat
5. API Key 是否正确。
6. DeepSeek 账户是否还有额度。
7. 如果出现 WinError 10013，通常是 Windows 防火墙、杀毒软件、代理/VPN 或当前运行环境禁止 Python 访问外网。请关闭当前服务窗口后，双击“启动实训报告网页.bat”重新启动；如果仍失败，请在防火墙中允许 python.exe 联网，并检查代理/VPN 设置。

如果仍然不能用，可以先关闭 AI 增强，用本地规则生成。
